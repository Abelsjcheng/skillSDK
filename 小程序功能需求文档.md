## 4. 详细功能需求（现状）

### EF-01 会话初始化

- 页面启动时从 URL Query 解析 `welinkSessionId`
- 参数缺失或非法时：
  - 页面记录错误状态
  - 停止后续会话初始化

### EF-02 JSAPI 调用约束

- 当前工程通过 `window.HWH5EXT` 调用如下接口：
  - `getSessionMessage`
  - `registerSessionListener` / `unregisterSessionListener`
  - `sendMessage`
  - `stopSkill`
  - `regenerateAnswer`
  - `sendMessageToIM`
  - `replyPermission`
  - `controlSkillWeCode`

### EF-03 首屏消息加载与监听

- 进入会话后先获取历史消息（`page=0,size=50`）
- 历史消息映射为前端统一消息结构后展示
- 完成加载后注册会话监听，持续接收增量消息
- 组件卸载时反注册监听

### EF-04 流式消息聚合与状态管理

- `StreamAssembler` 支持按 part 聚合：
  - `text.delta` / `text.done`
  - `thinking.delta` / `thinking.done`
  - `tool.update`
  - `question`
  - `permission.ask`
  - `file`
- 状态事件处理：
  - `session.status`：驱动 `idle/busy/retry`
  - `step.done`：更新 tokens/cost 元信息（App 形态）
  - `session.error` / `error`：错误展示与状态复位
  - `snapshot`：恢复历史快照（两种形态均有）
  - `streaming`：App 形态支持，组件库形态未处理

### EF-05 内容区渲染能力

- 消息气泡区分 `user` 与 `assistant`
- assistant 消息支持 Markdown（含表格、列表、引用）
- 代码块支持语法高亮与复制
- 支持 `onMessage` 返回的数学公式 Markdown 渲染：
  - 使用 `remark-breaks` 处理换行
  - 使用 `rehype-raw` 支持 Markdown 中内联 HTML
  - 使用 `remark-math` 解析行内/块级数学公式
  - 使用 `rehype-katex` 输出 KaTeX 公式渲染结果
- 流式消息显示光标动画
- 消息支持“复制”和“发送到 IM”操作按钮

### EF-06 场景化卡片渲染

- `ThinkingBlock`：思考过程折叠/展开
- `ToolCard`：工具执行状态卡片（pending/running/completed/error）
- `QuestionCard`：
  - 支持选项点击
  - 支持自定义输入
  - 提交时携带 `toolCallId` 调用 `sendMessage`
- `PermissionCard`：
  - 支持 `once/always/reject`
  - 调用 `replyPermission` 后切为已处理态
- `file`：文件链接展示

### EF-07 底部操作区状态机

- 当前按钮模式：`generate` / `generating` / `regenerate`
- 输入框显示规则：
  - `generate`：显示输入框 + “生成”
  - `generating`：隐藏输入框 + “停止生成”
  - `regenerate`：隐藏输入框 + “重新生成”
- 触发逻辑：
  - 点击“生成”：调用 `sendMessage`，输入框立即隐藏，按钮切换为“停止生成”
  - 点击“停止生成”：调用 `stopSkill`，按钮切换为“重新生成”
  - 点击“重新生成”：调用 `regenerateAnswer`，按钮切换为“停止生成”（重新进入生成中）
  - AI 执行完成并返回最终消息内容后：输入框重新显示，按钮恢复“生成”

补充说明（按 JSAPI 语义）：

- 输入框仅在“生成”按钮展示时可见
- “重新生成”必须调用 `regenerateAnswer` JSAPI，不走普通 `sendMessage`
- “AI 执行完成”以前端收到最终完成信号为准（如 `session.status=idle` 且当前流式消息已落定）

### EF-08 头部交互

- 缩小按钮：调用 `controlSkillWeCode({ action: 'minimize' })`
- 关闭按钮：调用 `controlSkillWeCode({ action: 'close' })`

### EF-09 发送到 IM

- 在内容区消息卡片右下角的“发送”按钮点击后，调用 `sendMessageToIM`
- 当前实现默认仅传 `welinkSessionId`（不显式传 `messageId`）

### EF-10 组件化导出与 require 示例

- 组件库通过 UMD 导出，不配置 externals，产物可在静态 HTML 直接使用
- 导出能力：
  - `AIChatViewer`
  - `mountAIChatViewer(container, props)`
  - `unmountAIChatViewer(container)`
- `example/require-demo` 已提供 `require('../../../dist/lib/index.js')` 的挂载、卸载、重挂载示例

### EF-15 数学公式 Markdown 渲染（需求补充）

- 适用范围：
  - 内容区普通文本 Part（`text`）
  - assistant/tool 消息 fallback Markdown 渲染
  - thinking 内容渲染
- 输入样式：
  - 行内公式：`$a^2+b^2=c^2$`
  - 块级公式：`$$\\int_0^1 x^2 dx = 1/3$$`
- 兼容要求：
  - 与现有 `remark-gfm`、代码块高亮能力共存
  - 公式渲染失败时不导致消息区域崩溃（至少保留原始文本展示）

## 5. 页面交互流程（现状）

1. 页面加载 -> 解析 URL 中 `welinkSessionId`
2. 调用 `getSessionMessage` 拉取历史
3. 注册 `registerSessionListener`
4. 用户输入并点击“生成” -> `sendMessage`
5. 监听 `onMessage` 实时更新内容区
6. 生成中点击“停止生成” -> `stopSkill`
7. 停止后可点击“重新生成” -> `regenerateAnswer`
8. 对权限卡片操作 -> `replyPermission`
9. 点击消息“发送” -> `sendMessageToIM`
10. 点击缩小/关闭 -> `controlSkillWeCode`

## 6. 现状约束与差异说明

- 监听注册时序现状为“先拉历史，后注册监听”，高并发场景存在窗口期漏消息风险
- `sendMessageToIM` 现状调用默认不传 `messageId`
- 组件库形态与页面形态在监听反注册参数、`streaming` 事件处理上存在细节差异
- `onError` 当前按 `{ errorCode, errorMessage }` 结构处理

## 7. 最小验收清单（现状）

1. URL 带 `welinkSessionId` 时可正常进入会话并加载历史消息
2. 点击“生成”后，用户消息入列，assistant 消息流式渲染
3. 点击“生成”后输入框隐藏，按钮切换为“停止生成”
4. 点击“停止生成”后按钮切换为“重新生成”，点击“重新生成”触发 `regenerateAnswer`
5. AI 返回最终消息并完成后，输入框重新显示，按钮恢复“生成”
6. 问题卡片可提交选项/自定义输入
7. 权限卡片可提交允许/始终允许/拒绝并成功落态
8. 点击内容区消息卡片右下角“发送”按钮可触发发送到 IM
9. 点击缩小/关闭可触发对应 JSAPI
10. 组件库产物可通过 require 方式挂载和卸载
11. `onMessage` 返回包含数学公式的 Markdown 时可正确渲染（含行内与块级公式）

## 8. 协议补充（结合 JSAPI 与服务端报文）

### 8.1 REST 包装与 JSAPI 返回关系

- `01-layer1-miniapp-skill-api.md` 中 REST 接口统一返回：
  - `code`：结果码，`0` 表示成功
  - `errormsg`：错误信息
  - `data`：业务数据
- `小程序JSAPI接口文档.md` 对外暴露的是业务层返回（等价于服务端 `data`），失败时走 Promise reject（`errorCode/errorMessage`）。

示例：

- 服务端 `sendMessage` 返回：`{ code, errormsg, data: { id, welinkSessionId, ... } }`
- JSAPI `sendMessage` 返回：`{ id, welinkSessionId, ... }`

### 8.2 JSAPI 与服务端报文映射（现状实现相关）

| JSAPI | 服务端参考 | JSAPI 入参 | 服务端请求关键字段 | JSAPI 返回 | 服务端 data 关键字段 | ai-chat-viewer 当前使用 |
| --- | --- | --- | --- | --- | --- | --- |
| `getSessionMessage` | `GET /api/skill/sessions/{welinkSessionId}/messages` | `welinkSessionId,page,size` | Query: `page,size` | `content,page,size,total` | `content[].id/role/content/messageSeq/parts[]` | 首屏拉取历史并映射为 `Message[]` |
| `sendMessage` | `POST /api/skill/sessions/{welinkSessionId}/messages` | `welinkSessionId,content,toolCallId?` | Body: `content,toolCallId?` | `id,welinkSessionId,userId,role,content,messageSeq,createdAt` | 同左 | 点击“生成”或问题回答时调用 |
| `regenerateAnswer` | SDK 二次封装（服务端参考为再次走 `messages` 发送） | `welinkSessionId` | 由 SDK 取最后一条用户消息后触发发送 | 与 `sendMessage`同构 | 与 `sendMessage`同构 | 点击“重新生成”调用 |
| `stopSkill` | `POST /api/skill/sessions/{welinkSessionId}/abort` | `welinkSessionId` | 无请求体 | `welinkSessionId,status` | `status=aborted` | 点击“停止生成”调用 |
| `sendMessageToIM` | `POST /api/skill/sessions/{welinkSessionId}/send-to-im` | `welinkSessionId,messageId?` | Body: `content,chatId?`（由 SDK 组装） | `status` | `success` | 气泡“发送”按钮调用，当前不显式传 `messageId` |
| `replyPermission` | `POST /api/skill/sessions/{welinkSessionId}/permissions/{permId}` | `welinkSessionId,permId,response` | Body: `response` | `welinkSessionId,permissionId,response` | 同左 | 权限卡片按钮调用 |
| `controlSkillWeCode` | 容器侧动作（非业务 REST） | `action: close/minimize` | - | `status` | - | Header 缩小/关闭调用 |
| `registerSessionListener` | `WS /ws/skill/stream` 推送 | `welinkSessionId,onMessage,onError,onClose` | 服务端主动推送 | 回调事件 | 见 8.3 | 内容区实时增量渲染 |

### 8.3 `registerSessionListener.onMessage` 报文基线（服务端 WS）

#### 传输公共字段

- `type`：事件类型
- `seq`：传输序号（单调递增）
- `welinkSessionId`：会话 ID（字符串）
- `emittedAt`：事件时间（ISO-8601）
- `raw`：原始事件（可选）

#### 消息层字段（消息相关事件）

- `messageId`
- `messageSeq`
- `role`

#### Part 层字段（分片相关事件）

- `partId`
- `partSeq`

#### 事件类型与当前前端处理

| type | 服务端关键字段 | 当前处理 |
| --- | --- | --- |
| `text.delta` / `text.done` | `partId,content` | 文本增量/完成 |
| `thinking.delta` / `thinking.done` | `partId,content` | 思考块增量/完成 |
| `tool.update` | `toolName,status,input,output,error,title` | 工具卡片状态更新 |
| `question` | `toolCallId,header,question,options` | 问答卡片渲染与回填 |
| `permission.ask` | `permissionId,permType,title,metadata` | 权限卡片渲染 |
| `permission.reply` | `permissionId,response` | 权限结果同步（已可接收） |
| `file` | `fileName,fileUrl,fileMime` | 附件展示 |
| `step.start` / `step.done` | `tokens,cost,reason`（done） | 会话中状态与元数据 |
| `session.status` | `sessionStatus` | 驱动 `busy/idle/retry` |
| `session.title` | `title` | 可选标题更新（当前未重点展示） |
| `session.error` / `error` | `error` | 错误展示与状态恢复 |
| `agent.online` / `agent.offline` | - | 预留状态处理 |
| `snapshot` | `messages[]` | 快照恢复（已实现） |
| `streaming` | `parts[],sessionStatus,...` | App 形态支持，组件库形态未处理 |

## 9. 错误码补充（JSAPI 文档口径）

高频错误码（按 `小程序JSAPI接口文档.md`）：

- `1000`：参数无效
- `4000`：会话不存在
- `4001`：会话已关闭（部分接口）
- `4002`：无用户消息（重新生成）
- `4003`：消息不存在
- `4004`：消息未完成
- `4005`：无最终消息
- `4006`：监听器不存在
- `4007`：权限请求不存在
- `4008`：权限请求已过期
- `4009`：小程序不存在
- `4010`：小程序操作失败
- `5000`：内部错误
- `6000`：网络错误
- `7000`：服务端错误
- `7001`：AI 网关错误

当前前端处理方式：

- Promise reject 后写入错误 Banner（或控制台日志）
- `registerSessionListener` 的 `onError` 事件也会进入错误展示通道

## 10. 语义说明（与现状实现强相关）

- `stopSkill` 语义是“中止当前执行”，不是关闭会话；会话通常仍可继续发送消息
- `sendMessageToIM` 的服务端实际请求体是 `content/chatId`，`messageId` 主要用于 SDK 本地定位要发送的最终内容
- 监听时序当前为“先拉历史后注册监听”，如需彻底避免窗口期漏消息，建议升级为“先监听后拉历史并做去重合并”
