# Android / Harmony / iOS 各接口入参出参与功能改动列表

基于最新 `01-layer1-miniapp-skill-api.md` 与 `SkillClientSdkInterfaceV1.md` 的对齐结果整理。

## Android

| 接口 | 入参改动 | 出参改动 | 功能改动 |
|---|---|---|---|
| `createSession` | `imGroupId` 必填 `string`；`ak/title` 可选 | `SkillSession.welinkSessionId` 为 `string` | 会话复用按 `imGroupId+status=ACTIVE`；有 `ak` 时附加 `ak` 条件；按 `updatedAt` 取最新 |
| `closeSkill` | 无 | `status` | 仅本地断开 WebSocket，不调用服务端删除会话 |
| `stopSkill` | `welinkSessionId: string` | `welinkSessionId: string` + `status=aborted` | 调 `/abort`，并触发 `stopped` 状态 |
| `onSessionStatusChange` | `welinkSessionId: string` + `callback` | 回调 `SessionStatusResult` | 监听 `session.status`，按 busy/retry/idle 映射执行中/完成 |
| `onSkillWecodeStatusChange` | `callback` | 回调 `SkillWecodeStatusResult` | 本地 wecode 状态通知 |
| `regenerateAnswer` | `welinkSessionId: string` | `SendMessageResult` 对齐文档结构 | 优先本地取最后一条用户消息，缺失再拉历史兜底 |
| `sendMessageToIM` | `welinkSessionId` 必填；`messageId/chatId` 可选 | `success: boolean` | `messageId` 仅本地定位不透传；`chatId` 仅透传；历史兜底；细分 `4003/4004/4005` |
| `getSessionMessage` | `welinkSessionId/page/size` | `PageResult` 用 `number/size/totalElements` | 服务端历史 + 本地流式缓存合并 |
| `registerSessionListener` | `welinkSessionId` + `onMessage`；`onError/onClose` 可选 | `status=success` | 同一 `welinkSessionId` 重复注册不覆盖、直接成功返回 |
| `unregisterSessionListener` | 仅 `welinkSessionId` | `status=success` | 按会话移除该会话监听 |
| `sendMessage` | `welinkSessionId/content` 必填，`toolCallId` 可选 | `SendMessageResult` 含 `seq/messageSeq/contentType/meta/parts` | 自动确保 WebSocket 连接并缓存用户消息 |
| `replyPermission` | `welinkSessionId/permId/response` | `welinkSessionId/permissionId/response` | 参数校验后透传服务端 |
| `controlSkillWeCode` | `action: close/minimize` | `status` | 本地状态通知并返回成功 |

## Harmony

| 接口 | 入参改动 | 出参改动 | 功能改动 |
|---|---|---|---|
| `createSession` | `imGroupId` 必填 `string`；`ak` 可选（空串不允许） | `SkillSession.welinkSessionId` 为 `string` | 会话复用查询逻辑对齐服务端（`imGroupId` + 可选 `ak` + `ACTIVE`） |
| `closeSkill` | 无 | `status` | 仅本地关闭 WebSocket |
| `stopSkill` | `welinkSessionId: string` | `welinkSessionId: string` + `status=aborted` | 调 `/abort` 并标记会话状态 |
| `onSessionStatusChange` | `welinkSessionId: string` + `callback` | 回调状态结果 | 状态映射与文档一致 |
| `onSkillWecodeStatusChange` | `callback` | 回调 wecode 状态 | 本地通知 |
| `regenerateAnswer` | `welinkSessionId: string` | `SendMessageResult` 对齐 | 缓存优先，历史兜底 |
| `sendMessageToIM` | `welinkSessionId` 必填；`messageId/chatId` 可选（空串按未传） | `success: boolean` | `chatId` 仅透传；`messageId` 本地查找；历史兜底；`4003/4004/4005` 细分 |
| `getSessionMessage` | `welinkSessionId/page/size` | `PageResult` 用 `number/size/totalElements` | 历史 + 本地缓存合并 |
| `registerSessionListener` | 同 Android | `status=success` | 同 session 仅注册一次，重复注册 no-op |
| `unregisterSessionListener` | 仅 `welinkSessionId` | `status=success` | 按 session 移除 |
| `sendMessage` | `welinkSessionId/content` 必填 | `SendMessageResult` 对齐 | 自动连 WebSocket，缓存用户消息 |
| `replyPermission` | `welinkSessionId/permId/response` | `welinkSessionId/permissionId/response` | 透传服务端 |
| `controlSkillWeCode` | `action` | `status` | 本地状态变更通知 |

## iOS

| 接口 | 入参改动 | 出参改动 | 功能改动 |
|---|---|---|---|
| `createSession` | `imGroupId` 必填 `NSString *`；`ak/title` 可选 | `SkillSession.welinkSessionId` 为 `NSString *` | 会话复用查询与文档一致（含可选 `ak` 条件） |
| `closeSkill` | 无 | `status` | 仅本地关闭 WebSocket，不调服务端删除 |
| `stopSkill` | `welinkSessionId: NSString *` | `welinkSessionId: NSString *` + `status` | 调 `/abort` 并触发 `stopped` |
| `onSessionStatusChange` | `welinkSessionId: NSString *` + 回调 | 回调状态对象 | 状态映射对齐 |
| `onSkillWecodeStatusChange` | 回调 | 回调 wecode 状态对象 | 本地通知 |
| `regenerateAnswer` | `welinkSessionId: NSString *` | `SendMessageResult` 结构对齐 | 缓存优先，历史兜底 |
| `sendMessageToIM` | `welinkSessionId` 必填；`messageId/chatId` 可选（空串按未传） | `SendMessageToIMResult.success (BOOL)` | `chatId` 仅透传；`messageId` 本地查找；历史兜底（`size=100`）；`4003/4004/4005` |
| `getSessionMessage` | `welinkSessionId/page/size` | `PageResult` 字段为 `number/size/totalElements` | 历史 + 流式缓存合并返回 |
| `registerSessionListener` | `welinkSessionId/onMessage` 必填 | `status=success` | 同 session 重复注册不新增监听，返回 success |
| `unregisterSessionListener` | 仅 `welinkSessionId` | `status=success` | 按 session 一次性移除监听 |
| `sendMessage` | `welinkSessionId/content` 必填；`toolCallId` 可选 | `SendMessageResult` 含 `seq/messageSeq/contentType/meta/parts` | 自动建连；发送结果入缓存并落 `seq` |
| `replyPermission` | `welinkSessionId/permId/response` | `welinkSessionId/permissionId/response` | 参数校验 + 透传 |
| `controlSkillWeCode` | `action` | `status` | 本地通知关闭/最小化 |

## 跨接口公共模型改动

1. 三端 `StreamMessage` 均补齐 `sourceMessageId` 字段并完成解析。
2. Harmony / iOS 本地消息排序统一为：`seq` 优先，`messageSeq` 次之，`id` 兜底。
3. `sendMessageToIM` 语义统一：`messageId` 仅用于 SDK 本地缓存定位，`chatId` 仅透传服务端。
